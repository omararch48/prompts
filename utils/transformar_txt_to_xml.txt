[PASOS A SEGUIR]

Ejecuta los siguientes pasos en orden y sin omitir ninguno:

1. Análisis de viabilidad: Evalúa si el prompt secundario es suficientemente claro para ser procesado. Si detectas ambigüedades críticas o falta de información que impidan ejecutar correctamente el prompt primario, detente aquí y solicita al usuario la información faltante antes de continuar.

2. Detección de idioma: Identifica el idioma del prompt secundario. El prompt resultante debe estar escrito en ese mismo idioma.

3. Correcciones ortográficas: Corrige todos los errores ortográficos presentes en el prompt secundario.

4. Anàlisis del flujo a seguir: Identifica los elemntos primarios del prompt en funcion del tipo de transformacion

4.1. De texto plano (txt) a XML: este caso lo determina el valor de tipo de transformacion "txt_to_xml"

4.1.1. Identificar en el prompt secundario que corresponde a un prompt txt, las secciones principales, subsecciones inmediatas y demas subsecciones anidadas, cada subseccion puede contener información como descripciones, listados, ejemplos etc

4.1.1. La estructura del prompt resultante (prompt XML) debera tener mediante etiquetas en orden logico, cada seccion y cada subseccion anidada respetando el orden indicado en el prompt secundario

4.1.2. Los contenidos que no sean subsecciones deben ser el contenido de la etiqueta XML a menos que las circunstancias ameriten que por cuestion de sintaxis XML, estos contenidos deban incluirse dentro de etiquetas XML, como por ejemplo, listas, textos de gran importancia, etc.

4.1.3. El contenido debe ser completamente equivalente, es decir si tomando un ejemplo, en el prompt primario se tiene la seccion "[REGLAS DE NEGOCIO]" que determina una seccion principal y esta al transformarse en una etiqueta XML deberia ser '<reglas_de_negocio section="REGLAS DE NEGOCIO"></reglas_de_negocio>'

4.1.4. Solo debe agregarse información que ayude a hacer mas entendible el prompt XML, como en el ejemolo anteriores '<reglas_de_negocio section="REGLAS DE NEGOCIO"></reglas_de_negocio>' se esta agregando un atributo llamado title que me permite volver a transformar de XML a texto, o por ejemplo en el caso de una lista txt con 3 elementos que se vea como 
```1.1. ...
1.1.1. ...
1.2 ...```
no vale la pena anidar listas, ya que es mas facil agregar informacion en un atributo 'tipo_lista="ordenada con anidacion"', pero en caso de que se tengan muchos elementos, el LLM debera determinar los casos en que es mas conveniente usar listas anidadas mediante etiquetas

5. Evaluación: Determianar si es que bajo esta nueva estructura la eficiencia del prompt, mejora, es equivalente o se reduce, escribir un resumen muy corto del analisis en pantalla y el analisis mas detallado debe escribirse en un archivo de texto plano argumentando el resultado

6. Generación del prompt resultante: Produce el prompt resultante con los cambios aplicados en los pasos anteriores.
